Tokens

Design tokens are the single source of truth for every visual property in Lamplight. They are emitted from Figma via Style Dictionary and consumed as CSS custom properties.

Three-layer architecture

Each layer adds semantic meaning. Only the bottom layer is consumed by components and application code.

Layer 1

Brand

Raw primitives exported from Figma — color ramps, spacing scales, font families, and z-index values.

Token authors and design tooling only. Never reference directly in components.

  • --brand-brand-600
  • --brand-scale-400
  • --brand-font-family-body
→
Layer 2

Alias

Semantic names that map brand primitives to intent — primary, neutral, error, spacing, and radius.

Spacing, radius, and elevation in components. Color aliases inform mapped tokens.

  • --alias-primary-600
  • --alias-spacing-lg
  • --alias-border-radius-md
→
Layer 3

Mapped

Component-facing tokens with light and dark values. This is the layer every UI surface consumes.

All component and app code. Use --color-*, --spacing-*, --border-radius-* only.

  • --color-surface-action
  • --color-text-body
  • --spacing-xl

Do

Reference --color-*, --spacing-*, and --border-radius-* in component code.

Don't

Reference Brand tokens like --brand-brand-600 directly in components, or hardcode hex values.

Spacing

Spacing tokens follow a 4px base grid. Use Alias spacing tokens in all padding, margin, and gap values.

--spacing-xs

4px

Tight internal padding

--spacing-sm

8px

Compact padding, icon gaps

--spacing-md

12px

Default component padding

--spacing-lg

16px

Gaps between elements

--spacing-xl

24px

Section-internal spacing

--spacing-2xl

32px

Between sections

--spacing-3xl

48px

Page-level spacing

Border radius

Radius tokens define corner rounding for inputs, cards, modals, and pill-shaped elements.

TokenValueUse
--border-radius-none0Square corners
--border-radius-sm2pxSubtle rounding
--border-radius-md4pxInputs, cards
--border-radius-lg8pxModals, larger panels
--border-radius-xl16pxFeature cards
--border-radius-full999pxPills, badges, buttons

Elevation

Four shadow utilities map to Figma effect styles. Pair raised surfaces with shadow-raised, overlays with shadow-overlay.

TokenUse
shadow-raisedRaised — Cards lifted above the default surface
shadow-overlayOverlay — Dropdowns and popovers
shadow-modalModal — Dialogs and modals
shadow-toastToast — Notifications and toasts

Z-index

Layering tokens keep overlays, modals, and toasts in predictable stacking order.

TokenValueUse
--brand-z-index-dropdown100Dropdown menus
--brand-z-index-sticky200Sticky headers and sidebars
--brand-z-index-overlay300Modal backdrops
--brand-z-index-modal400Modal content
--brand-z-index-popover500Popovers and tooltips
--brand-z-index-toast600Toast notifications
